## 二、整体数据流

```
红外热像仪 ──GPIO──→ 树莓派 ──MQTT──→ 后端 Django ──SSE──→ 前端 Vue 3
                            ←──MQTT──  (控制指令)
```

| 环节 | 说明 |
|------|------|
| 树莓派 → 后端 | MQTT 推送温度数据、红外图像帧、告警通知、断路器状态 |
| 后端 → 树莓派 | MQTT 下发分闸/合闸/复位控制指令 |
| 后端 → 前端 | SSE 推送实时状态（每5秒）、告警（即时）、红外图像帧（每秒1帧） |
| AI 推理 | 后端的红外图帧经过 ONNX 模型推理 → 产出热斑标注图 → SSE 推前端 |

## 四、数据关系

```
后端 A (GPU)                   后端 B (无GPU)                   前端
───────────                    ──────────────                  ──────
训练 UNet ──┐
            ├──→ ONNX 模型 ──→ 嵌进 SSE image 管道 ──→ ThermalImg 展示标注图
            │
            ├──→ 分割精度 ────→ 填进 /api/ai/metrics ──→ PerformanceMetrics 展示
            │
            └──→ 模型信息 ────→ 填进 /api/ai/model-info ──→ ModelInfo 展示

后端 B 自己：
  Mosquitto ──→ /api/mqtt/status ──→ MqttStatus 连接状态
  MQTT 消息 ──→ SSE /api/mqtt/messages ──→ MqttStatus 实时日志
```
